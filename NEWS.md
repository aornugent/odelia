## Odelia 0.0.0.9000

* New self-contained example `soil_leaf` (`inst/examples/soil_leaf_interface.cpp`, `test-ad-soil-leaf.R`): the plant-free witness for the coupled-dynamics pieces the static-leaf `weibull_leaf` cannot exercise. A soil-water ODE (per-layer `θ_i`) whose potential `ψ_soil,i = a(θ_sat/θ_i)^n` drives the toy leaf's per-layer uptake `E_i = kmax(G(p) − G(ψ_soil,i))` through the same `incomplete_gamma` hydraulics; that uptake is the soil sink `dθ_i/dt = (inflow − ΣE_i)/dz`, closing the two-way feedback loop. Integrated through the odelia `Solver`, with a `ci` `implicit_value` node **inside** `ode_rates` (a leaf node living in the ODE) and a consumer introduced mid-run (the growing tape / resize). Reverse-mode `d(final biomass)/d(kmax, c)` matches a re-integrating finite difference to <1e-9 (the `c` channel flows entirely through the feedback via `incomplete_gamma`'s shape derivative), the active value reproduces the double run bit-for-bit, and the whole run's tape stays a few MB (linear, not the OOM blow-up). Confirms the soil sub-cycle adjoint, node-in-rates, and resize are correct together — standalone, before the plant wiring.

* `odelia::implicit_value` gains an optional `denom_sign expect` argument that asserts the sign of `dF/dy` at the operating point (`positive`/`negative`/`any`, default `any`): at a fold `dF/dy → 0` the node would divide by ≈0 and return a silent garbage gradient, so declaring the sign turns that into a loud stop. `odelia::register_implicit` — the explicit-input-vector, forward-mode sibling — is **removed**: it had no production callers, and its one distinguishing feature (the sign guard) now lives on `implicit_value`. The two were not merged (their mechanisms differ; a merge would only relocate the switch). The `implicit_node` example/test now exercise `implicit_value` across double/forward/reverse plus the guard.

* New self-contained example `weibull_leaf` (`inst/examples/weibull_leaf_interface.cpp`, `test-example-weibull-leaf.R`): a plant-free miniature of the `plant`/TF24 leaf built only from odelia primitives — `incomplete_gamma` Weibull hydraulics, a nested inner `ci` root (`implicit_value`), and an outer collar-tension optimum `p*` (`implicit_value` on the stationarity residual). It certifies reverse-mode AD against a re-optimising finite difference and covers all three of the composition's hard cases, independent of plant:
  - **Interior optimum** (`weibull_leaf_demo`): four trait channels, separating the two envelope facts — `dW/dθ` at the optimum is the direct partial only (the `dp*` term vanishes — envelope theorem), while `d(E_up)/dθ` carries `dp*/dθ` (envelope asymmetry).
  - **Bound / fold regime** (`weibull_leaf_bound_demo`): the leaf pinned at a hydraulic bound `p_crit` solved by a *branch-death* `implicit_value` (residual `cond(p) − k_crit`, regular denominator). Here `dW/dp ≠ 0`, so — unlike the interior case — the profit gradient itself carries `dp*/dθ`; both are checked against FD.
  - **Two-layer soil feedback** (`weibull_leaf_soil_demo`): per-layer uptake `E_i(p) = k_i(G(p) − G(ψ_i))` with the layer-crossing breakpoint, summed into `gc`; the spatial `ψ_0/ψ_1` channels and per-layer sinks `dE_i/dθ` (the historic sign-error site) are all FD-verified and sign-checked.
  - **Full 3-deep nest** (`weibull_leaf_stem_demo`): the transpiration-inversion `ψ_stem` restored between the collar optimum and the `ci` root — a second `implicit_value` (inversion, regular sign-definite denominator) so the chain is `p → ψ_stem → ci`. `d(ψ_stem)/dθ`, `dp*`, and `dprofit` all FD-verified across the four traits.
  - **Resistance-network uptake** (`weibull_leaf_uptake_network_demo`): the *real* `leaf_output::soil_uptake` arithmetic — a Darcy flux `E_i = (ψ_soil,i − P_x_r − z_i)/(area·r_R,i)` through a resistance network whose `r_R` divides by the mean fractional conductivity (a `cumulative_vuln`/`incomplete_gamma` integral, split at zero), with the two degenerate potential-equality guards. This is the surface the clean `kmax(G(p)−G(ψ))` demos never exercised, and it carries the one channel the S-signature widening exists to restore: the root Weibull shape `root_b`/`root_c` is **seeded** and reaches `E_up` only through that integral, so `d(E_up)/d(root_c)` — via the series `d/dc` — was structurally zero before. Every channel (incl. `root_b`/`root_c` and both per-layer sinks) FD-matches.
  - **Bound as a continuity residual** (`weibull_leaf_bound_continuity_demo`): p\* pinned where the network supply meets a fixed demand `E_up(p) − demand = 0` (TF24's `E_column` bound), the resistance-network uptake inside the residual, regular sign-definite denominator `dE_up/dp > 0`. `dp*/dtrait` FD-matches.
  - **Value-graft guard** (`weibull_leaf_graft_demo`): TF24 replaces the assembled S value with the separately-converged double leaf value (`anchor(leaf.profit_, profit_assembled)`), keeping only the derivative. Witnesses that at the correct inner anchor the raw assembled value equals the double truth (the precondition that makes grafting safe), that a wrong anchor makes them diverge (the `raw == truth` guard discriminates), and that grafting the truth value onto the assembled derivative leaves the gradient equal to the ungrafted FD gradient.
  - **Tape-memory bound** (`weibull_leaf_tape_profile`): the load-bearing OOM proof. The `implicit_value` (solve-off-tape) path records a tape whose size is **exactly independent of the inner solver's iteration count** and ~1.6k ops per leaf solve; a naive on-tape solve grows with iterations and costs ~26× more per solve (and, being a frozen-bracket record, returns a *wrong* zero gradient). Confirms the design bounds the tape that the status-quo TF24 blows up.
  Serves as the durable regression witness that the primitive stack composes correctly — and stays memory-bounded — across the interior, fold, feedback, and full-nest cases.

* `basic_interpolator::slope(u, step, direction)`: the field-slope `d(value)/du` by **secant** over `step` (backward/centred/forward), the robust rate-path derivative channel and the single owner of that quantity (#38 / plant#39, design B). Unlike `deriv()` (the analytic tangent, which is unreliable for an under-resolved fit and compounds on an evolving-state query), the secant is exactly the finite difference a caller's own value stencil takes, so a consumer's slope matches that stencil by construction when it passes the same step + direction. Query values frozen, knot-value (parameter) derivatives carried, nesting-safe (`FReal<AReal<double>>`). A downstream System that needs `dS/dx` on a rate path reads this rather than hand-rolling a secant (retires plant's `dg/dh`-coupling triplication).

* New `odelia::ad::compute_jvp` / `compute_directional_derivative` (`gradient.hpp`): the forward-mode (tangent) Jacobian-vector product of an ODE solve — one augmented run, **no tape**, and transparent across a growing-dimension resize (nothing to register/reserve/keep-alive, unlike the reverse path). Paired with the reverse `compute_gradient` it gives the **FD-free adjoint dot-product oracle** `⟨J v, u⟩ = ⟨v, Jᵀ u⟩` (one forward + one reverse pass, no perturbation, no inner-solve re-run); exercised on a growing two-parameter System with a closed form (`inst/examples/jvp_oracle_interface.cpp`, `test-ad-jvp-oracle.R`). Runs the System at the single forward layer `FReal<double>`.

* New `odelia::ad::directional_derivative` (`directional_derivative.hpp`): a forward-over-reverse directional derivative that stays active on an outer reverse (adjoint) tape (#38). It computes `df/dx` along one seeded direction exactly — the analytic derivative of the code, with clamps/kinks handled correctly — so an enclosing reverse sweep differentiates it further into a mixed second derivative with no finite-difference step. The working type composes as `FReal<AReal<double>>` (tangent-over-adjoint, #35). Intended for terms like plant's density-transport `dg/dh` evaluated inside a parameter reverse sweep (plant#39).

* `basic_interpolator` active-query reads: the **default** `eval(Q)` / `operator()(Q)` now **freeze** the query-point derivative (read at the fully-stripped query value) while still carrying the knot-value derivatives — the safe rate-path read, value-identical to a double read (#38). A caller that genuinely wants `d(value)/d(query)` (a well-resolved spline queried at a real differentiation input) must opt in explicitly via the new `eval_with_query_derivative(Q)`. The analytic tangent of an under-resolved spline is unreliable for an evolving ODE-state query and compounds across a time integration (the bug behind plant#39); making it non-default means it can never be reached by accident.

Odelia is a new package, arsising out of https://github.com/traitecoevo/plant/. In that project, Rich FitzJohn built a custom ODE solver, using Runge-Kutta4-5 method, in C++. I'm spinning that code out into a package, as I want to use it elsewhere. 

* New implicit, adaptive-step **RODAS4(3)** Rosenbrock stepper for stiff systems, selectable via `method = "rodas"` when constructing a solver (#35). It reuses the existing adaptive step-size controller and obtains an exact Jacobian by forward-mode automatic differentiation; systems opt in by providing a `template<class U> rebind<U> rebind_from()` method (the same double->AD lift the gradient driver uses). The explicit RKCK 4(5) method (`method = "rkck"`) remains the default.

* `odelia` now loads its shared library with global symbol visibility in `.onLoad`, so packages that `LinkingTo: odelia` and instantiate `Solver` can resolve the compiled XAD runtime symbols at load time without per-package linker hacks (#26).
